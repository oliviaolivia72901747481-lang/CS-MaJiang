# 长沙麻将多人联网 v0.8.1 手动验收 Checklist

本 Checklist 用于指导局域网/手机热点多端测试，验证联网稳定性、重连恢复、重复点击防护及暗牌隔离。

## 一、网络环境与服务启动
- [ ] 1. **手机热点建局域网**：房主电脑连接手机热点或在同一路由器 WiFi 局域网下。
- [ ] 2. **服务端启动**：房主电脑在项目根目录下启动 Socket.IO 服务器：
  ```bash
  npm run server
  ```
- [ ] 3. **前端多端启动**：房主电脑启动 Vite 服务并允许局域网设备接入：
  ```bash
  npx vite --host 0.0.0.0 --port 5173
  ```
  *(记录控制台输出的局域网 URL，例如 `http://192.168.1.100:5173`)*

## 二、多端创建与加入房间
- [ ] 4. **手机 A 访问**：手机 A 访问 `http://房主IP:5173`，切换为“多人联机模式 (v0.8)”，输入昵称。
- [ ] 5. **手机 B 访问**：手机 B 访问 `http://房主IP:5173`，切换为“多人联机模式 (v0.8)”，输入昵称。
- [ ] 6. **手机 A 创建房间**：手机 A 点击“创建新对局”创建房间，显示 6 位数字短码（例如 `298103`）。
- [ ] 7. **手机 B 加入房间**：手机 B 输入房间号 `298103`，点击“加入”。此时大厅展示两位玩家名称。
- [ ] 8. **机器人补位**：点击“机器人补位”填满空闲座位（填充为 Advanced Lite AI 机器人）。
- [ ] 9. **开始游戏**：点击“开始游戏”启动 authoritative state machine，进入牌桌。

## 三、对局同步与动作校验
- [ ] 10. **A 出牌**：轮到 A 时，手机 A 点击手牌出牌。
- [ ] 11. **B 页面同步**：手机 B 界面看到 A 出牌日志及牌桌丢弃区立即同步更新。
- [ ] 12. **防重复点击**：手机 A 或 B 在出牌/吃碰杠胡过动作时，**快速连续双击或多次重复点击**，服务端在 `game-session.ts` 的 `action-dedupe` 防护下，只执行第一次，其余重复点击均返回 `ignored`，界面无任何卡死或多张手牌飞出。
- [ ] 13. **非法动作拒绝**：尝试通过修改前端变量或伪造包提交不属于自己当前手牌的 `tileInstanceId`，服务端 action guard 应直接拒绝并返回 error，不改变 `GameState`。

## 四、刷新重连与断线恢复
- [ ] 14. **B 刷新页面**：手机 B 在游玩中强制刷新浏览器页面。
- [ ] 15. **B 自动重连**：刷新加载后，前端 Hook 从 `localStorage` 读取 token 并自动发送 `game:sync` 进行会话同步，手机 B **直接重回座位**，无任何报错或被阻挡在外的提示。
- [ ] 16. **暗牌安全隔离**：手机 B 重连后，手机 B 依旧**只能看到自己的手牌**，查验控制台 HTTP/WS 数据流中，绝对不包含任何其他手机 A 或 AI 的暗手牌 `instanceId`。
- [ ] 17. **断网标记**：关闭手机 A 的 WiFi/蜂窝网，此时手机 B 及电脑浏览器控制台能看到手机 A 状态变为 `🔴 离线`。
- [ ] 18. **恢复重连**：重新开启手机 A 网络，A 页面刷新或自动重连回局，座位恢复为 `🟢 在线`。

## 五、对局结束与结算
- [ ] 19. **打满一局**：正常轮流推进，直至有人胡牌或摸完牌墙进入流局。
- [ ] 20. **公开手牌结算**：结算页弹出，此时且仅在此时，view 会 reveal 所有人暗手牌，手机 A 结算弹框中正确渲染出手机 B、AI 1、AI 2 局终时的所有手牌明牌。
- [ ] 21. **无崩溃与未捕获异常**：整个流程控制台及后台 Terminal 无任何 crash/Uncaught Exception 错误。
