## 一、本次修改目标
本次优化旨在解决以下两个桌面端的核心体验痛点：
1. **吃、碰、杠时关联牌不够醒目**：玩家难以看清自己的哪些手牌参与了动作、哪个是触发本次操作的来源牌，以及在多种候选组合时没有做联动筛选。
2. **桌面端页面高度溢出，产生纵向滚动条**：原先的弃牌河分散在各个玩家卡片中，撑大了界面，导致在 1366×768、1440×900 等桌面分辨率下无法一屏完整显示牌桌和核心操作区。

## 三、吃碰杠关联牌高亮实现说明
- **手牌动作高亮**：在玩家面临吃、碰、杠选择时，手牌中可以使用的候选牌组合会根据操作类型获得不同的外边框发光和高亮效果（吃：蓝色；碰：金色；杠：绿色）。
- **多选项联动 (Hover Filtering)**：如果吃或杠有多个组合，展示多选浮层。鼠标悬停在浮层中某一个组合上时，手牌中仅高亮属于该组合的牌，其余牌变回普通状态；移开后恢复全亮。这极大地防止了出牌混乱。
- **来源牌高亮**：触发本次吃碰杠的他人弃牌（来源牌），在中央弃牌区会伴有红色闪烁外框高亮显示 (`highlight-source`)，指向明确。
- **副露徽标与最新副露强调**：完成动作后，副露区对各组副露绘制专属彩色边框并配有文字徽标（「吃」「碰」「明杠」「补杠」「暗杠」）。最新完成的副露伴有 1.8s 短暂放大及金光呼吸效果，易于识别。

## 四、桌面端布局优化说明
- **一屏全显 (100vh 锁定)**：整个桌面端页面容器高度被强制限制为 `100vh` 且设置了 `overflow: hidden`，彻底排除了浏览器纵向滚动条。
- **中心弃牌区 (Unified Discards)**：移除了各玩家独立分散的 Discard 区域，统一整合成桌面中央 `(2, 2)` 单元格的 **Center Discard Area**，弃牌按行归拢。各行在人数变化（2/3/4人）时动态隐藏 inactiveSeats，不渲染无关玩家。最新打出的牌金色高亮闪烁，清牌方便。
- **对局日志栏**：独立于主对局进行高度自适应内部滚动，在各种小分辨率下完全不撑大页面高度。
- **自适应间距与排版**：减少了对手卡片的占用高度，为手牌区和操作按钮留出空旷视野。

## 五、安全隐私与手机端兼容性回归
- **暗牌安全**：继续严格遵守 `server-visible-view.ts` 规定，不泄露对手暗手牌和牌墙。对手的暗杠副露，只展示首尾两张，中间两张仍然做背面隐藏处理，杜绝作弊。
- **手机端回归**：本优化只在 width > 768px 时对电脑版生效，手机端依然完全保留 v0.8.4-hotfix 成果，不受任何排版或杠操作的影响。
