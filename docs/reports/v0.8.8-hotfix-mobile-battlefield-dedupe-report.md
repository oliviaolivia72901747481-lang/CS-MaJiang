# v0.8.8 Hotfix: Mobile Battlefield Dedupe

## Problem

The mobile battle screen still had duplicated battlefield information after the compact layout pass. The independent latest-discard dock and standalone action-source card consumed vertical space and repeated data already available in the center discard river and action candidates.

## Changes

- Removed `MobileLatestDiscardDock` from `MobileOnlineGameLayout`; mobile now relies on `MobileCenterDiscardArea` for all latest-discard context.
- Added a per-seat latest-discard red dot in the center discard river using `.tile-seat-latest-dot`.
- Kept the global latest-discard emphasis with `.discard-tile-latest.tile-global-latest-discard`.
- Removed the standalone mobile `ActionSourceTileBanner`.
- Added `compactTitle` support to `ActionCandidatePanel`, e.g. `下 打出 3条, 可碰 / 可胡`.
- Replaced action-source player names with relative labels (`我/上/下/对`) and guarded invalid source seats.
- Preserved chi/peng/gang/hu candidate groups and the pass button.
- No Mahjong rule, server, desktop layout, trustee, or hidden-information logic was changed.

## Modified Files

- `src/changsha-mahjong-network/components/MobileOnlineGameLayout.tsx`
- `src/changsha-mahjong-network/components/MobileCenterDiscardArea.tsx`
