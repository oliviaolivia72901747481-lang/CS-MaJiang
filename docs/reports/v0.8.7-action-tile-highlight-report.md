# 版本发布报告: 长沙麻将 吃碰杠关联牌特殊渲染高亮系统 (v0.8.7)

本报告详细说明了长沙麻将吃、碰、杠高亮系统的设计逻辑、具体修改方案、测试覆盖情况及版本兼容性保障。

---

## 一、高亮系统功能概述

长沙麻将 v0.8.7 引入了**动作关联牌特殊高亮渲染系统**，通过对“动作响应来源”、“参与手牌组合”、“候选牌型展示”和“新形成副露”进行层次分明的视觉增强，极大地提升了玩家在关键决策阶段的信息获取效率与可读性：

1. **响应来源牌高亮**：
   - 别人打出的牌若触发了我的吃、碰、直杠等挂起动作，该牌会被赋予 `tile-action-source` 样式，并在提示条（`ActionSourceTileBanner`）中明确标注出牌人（例如 `响应来源: 玩家2 打出 [7筒]`）。
2. **参与手牌高亮**：
   - 当前吃碰杠动作选项所需消耗的手牌会被赋予 `tile-hand-participant` 样式，产生呼吸闪烁效果，指示玩家该项操作会合并哪些手牌。
3. **候选组合面板**：
   - 候选组合列表（`ActionCandidatePanel`）直观呈现每个备选方案的牌面组合（如吃牌的不同搭子组合），当玩家在桌面端 hover 特定组合时，对应手牌的高亮状态会动态更新。
4. **新形成副露高亮**：
   - 玩家做出吃、碰、杠响应后，新生成的副露在桌面端与移动端均会应用 `.meld-newly-formed` 闪烁动效，持续 1.5 秒，引导视觉焦点。

## 二、关键设计原则与问题解答

按照本次开发补充约束，我们在设计中严格贯彻了以下几点：

### 1. 保留原有 `performAction` Payload 与动作流
- 本系统定位为**视觉增强层（Visual Enhancement Layer）**而非规则校验层。
- 吃、碰、杠、过等动作的提交负载（`payload`）与原规则引擎完全对齐：
  - 吃（`chi`）：`{ type: 'chi', optionId: '...' }`
  - 碰（`peng`）：`{ type: 'peng' }`
  - 杠（`gang`）：`{ type: 'gang', gangType: '...', tileKey: '...' }`
  - 过（`pass`）：`{ type: 'pass' }`
- **多个吃法的提交区分**：在 `action-highlight-utils.ts` 中，我们为每一个吃牌选项分配了基于该组合排好序的 tile key 字符串（如 `6wan,7wan,8wan`）作为 `optionId`。点击对应的候选组合按钮时，直接将该 `optionId` 作为 payload 传给 `performAction`，确保了动作指向的精确性。

### 2. 完整保留“过/Pass”按钮与响应能力
- 候选组合面板与“过/取消”动作并存，不论桌面端（`OnlineGamePage.tsx`）还是移动端（`MobileOnlineGameLayout.tsx`），玩家均可随时点击右侧独立的“过”按钮拒绝响应。
- 绝不遮挡、修改或跳过任何原有的“过”逻辑。

### 3. 暗杠与补杠不显示“响应来源”
- **暗杠 (anGang)** 和 **补杠 (buGang)** 是基于玩家自己摸到的牌或已有副露主动发起的操作，不需要其他玩家的弃牌作为媒介。
- 因此在 `action-highlight-utils.ts` 的模型构建中：
  - `anGang` 与 `buGang` 产生的 `ActionCandidateGroup` 不包含 `sourceSeat`、`sourceTile` 或 `sourceTileKey` 属性。
  - `ActionSourceTileBanner` 在 `sourceEvent` 为 `null` 时自动返回 `null`（不进行渲染），完全符合“无响应来源时不显出响应框”的合理预期。

### 4. 移动端非 hover 交互设计
- 移动端去除了 hover 依赖。吃碰杠等候选选项以卡片组合形式在 `ActionCandidatePanel` 中平铺展示，牌面清晰易点。
- 每个候选卡片具有触控优化样式（`min-height: 44px`），用户直接点击即可执行相应动作。

### 5. 新形成副露高亮的低风险设计 (零 Hooks)
- 在对 `MobileMeldArea.tsx` 的测试中，项目原有用例采用纯函数调用的形式来测试渲染输出。如果在该叶子组件中使用 `useState` / `useEffect` / `useRef`，会在测试环境下由于缺乏 React 渲染上下文而直接崩溃。
- 为了零风险地完成新形成副露的高亮动作，我们去除了所有 React hooks，仅在 loop 渲染时，对倒数第一个 meld 节点（即 `idx === melds.length - 1`）赋以 `.meld-newly-formed` 类名。
- 利用 CSS `@keyframes meld-flash` 挂载即播的特性，当新副露节点在 DOM 挂载时，浏览器会自动触发 1.5 秒的脉冲动画，既达到了高亮动效，又保持了对原有测试用例 100% 的兼容性。
