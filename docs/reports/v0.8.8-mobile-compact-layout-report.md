# v0.8.8 移动端紧凑布局报告

## 1. 目标
本次版本聚焦移动端四人联机对局的纵向空间压缩。目标是在不改动规则引擎、服务端状态推进、AI 托管、隐藏信息隔离和桌面核心布局的前提下，让手机首屏能同时保留对手信息、最近弃牌、中心弃牌河、操作提示、手牌和次级工具入口。

## 2. 初始高度来源
检查发现移动端高度主要消耗在三类区域：对手条中的长昵称和完整状态文案、最近弃牌/中心弃牌河的重复玩家名和较大牌面、底部常驻日志/诊断/帮助/离开按钮。顶部全局模式切换在进入房间 URL 时已可隐藏，但从大厅创建房间进入对局时仍会保留。

## 3. 保护边界
本次只修改展示层和测试。未调整 `RoundController`、规则判定、socket 房间生命周期、AI 出牌/托管逻辑、服务端私有视图生成，也未改变桌面端对局主布局的数据流。

## 4. 相对座位标签
新增 `getRelativeSeatLabel()`，将座位转换为 `我 / 下 / 对 / 上`。该工具支持 2 人、3 人和 4 人活跃座位，并对非活跃座位抛错，避免静默显示错误标签。

## 5. 最近弃牌区
`MobileLatestDiscardDock` 改为短标题“最近”，玩家名显示相对标签，真实玩家名保留在 `title`。最近牌使用 `compact` 尺寸，条目横向不换行，避免长昵称撑高区域。

## 6. 中心弃牌河
`MobileCenterDiscardArea` 改为最多两列网格。每位玩家只显示最近 2 张弃牌，更多历史以 `+N` 徽标进入历史抽屉；空弃牌状态保留“未出牌”。玩家标签同样改为相对标签。

## 7. 副露与对手条
`MobileMeldArea` 支持更小的 `mini/compact` 牌面。对手条压缩 padding、字号和手牌文案，同时保留原有连接状态语义，例如“重连中...”和“已离开 (AI托管)”。

## 8. 操作提示压缩
移动端出牌提示由长句压缩为“你出牌”和“再点确认出牌”。“我的副露”标题移除，副露内容仍保留在手牌上方。

## 9. More 抽屉
底部只常驻“更多”按钮。对局日志、局域网诊断、帮助和离开房间移动到抽屉中。离开按钮仍保留确认流程：对局中提示退出后 AI 托管，结算阶段提示离开结算页。

## 10. 模式切换隐藏
`MainApp` 新增 `shouldHideModeSwitch()`，在房间 URL 或联机对局页活跃时隐藏全局模式切换。`OnlineLobbyPage` 只上报是否正在渲染对局页，不改变联机状态流。

## 11. 新增测试
新增 `relative-seat-label.test.ts` 覆盖 2/3/4 人相对座位映射和非法座位。新增 `mobile-compact-layout.test.tsx` 覆盖模式切换隐藏、相对标签、紧凑弃牌尺寸、More 抽屉和冗余提示移除。

## 12. 回归测试更新
更新移动端弃牌可见性、中心弃牌层级、离开流程、移动端在线布局和弃牌去重相关测试，使其匹配新的网格结构和 More 抽屉入口。旧的连接状态文案断言继续通过。
