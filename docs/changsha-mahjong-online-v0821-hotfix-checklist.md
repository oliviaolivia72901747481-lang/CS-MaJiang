# 长沙麻将扫码加入与手机端界面热修复 v0.8.2.1 手动验收 Checklist

本验收单用于验证扫码联机加入的真实 IP 跳转能力和手机端精简牌桌的实际可用性。

---

## 💻 房主端（电脑端）验证
- [ ] **1. 一键启动命令**
  - 在电脑终端运行 `npm run dev:online`，检查控制台是否输出了正确的局域网 IP（而不是 localhost）。
- [ ] **2. 房主进入大厅**
  - 房主打开 `http://localhost:5173` 创建房间。
- [ ] **3. 二维码 localhost 警告检测**
  - 房主刚创建房间时，若默认链接是 localhost，界面会显示黄色警告提示：“警告: 手机不能使用 localhost，请在下方选择局域网 IP 链接生成二维码”。
- [ ] **4. 切换局域网 IP 候选**
  - 房主在页面下方的“局域网联机二维码地址切换”列表中选择标注“★ 推荐手机扫码使用”的局域网 IP。
  - 确认点击后，分享的二维码图片刷新，并且二维码下方的加入链接更新为：
    `http://房主局域网IP:5173/?mode=online&roomId=XXXXXX`。

---

## 📱 手机玩家端（微信或手机浏览器）验证
- [ ] **5. 手机扫码或访问局域网链接**
  - 手机 A 扫描推荐的局域网二维码，跳转到 `http://房主局域网IP:5173/?mode=online&roomId=XXXXXX`。
- [ ] **6. 自动进入多人联机模式 & 房间号自动带入**
  - 手机页面加载后，自动跳过首页选项，直接展现“多人联机大厅”昵称框。
  - 页面中的房间号框内已自动填充房主的 6 位房间号。
- [ ] **7. 历史昵称自动填充**
  - 若该手机之前玩过该游戏，其历史昵称会自动显示，并展示“自动加入上次使用的昵称”复选框。
- [ ] **8. 成功加入与 Socket 域名绑定**
  - 输入昵称后点击加入，手机成功加入席位，并在“局域网自检诊断面板”里确认：
    - Socket 真实连接地址为 `http://房主局域网IP:3001`（绝对没有 localhost 硬编码！）。
    - HTTP health 为 `健康 (ok)`。

---

## 🀄 手机端游戏牌桌（手机浏览器宽度 <= 768px）验证
- [ ] **9. 无横向滚动条**
  - 手机页面在各种竖屏模式下均无横向溢出，无法向右滑动。
- [ ] **10. 压缩的对手信息条**
  - 上方显示三个精简的对手信息条（包括：座号、昵称、在线状态、手牌数、吃碰杠明牌河）。
- [ ] **11. 中心极简牌桌**
  - 仅显示当前轮次玩家、剩余牌数以及最新打出牌的卡片（不显示巨大的空洞和无用间距）。
- [ ] **12. 动作按钮尺寸**
  - 出现吃/碰/杠/胡/过选项时，按钮在自己手牌上方悬浮排列，高度大于等于 44px，拇指非常容易点击且不遮挡任何手牌。
- [ ] **13. 固定底部手牌**
  - 自己的手牌固定贴合在屏幕底部，牌宽约 30px，选中出牌触感准确灵敏。
- [ ] **14. 对局日志、诊断、帮助默认折叠**
  - 这三个面板在游戏开始时均处于折叠状态，不占用任何牌桌空间。
  - 点击“📖 查看对局日志”按钮会滑出底部半透明抽屉，关闭后抽屉收回。
- [ ] **15. 对局结算回归**
  - 结算页面在手机端能够纵向滚动查看，且能够清晰辨认四家暗手牌的 reveal 状态，点击“返回大厅”按钮大小合适。
