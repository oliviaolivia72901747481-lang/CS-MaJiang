# 长沙麻将多人联网版 v0.8.2.2 退出机制与掉线重连验收单

本单列出了在手机热点/局域网及本地开发环境下，对玩家主动退出、被动断线、断线重连、托管接管、席位释放以及房间定时清理等逻辑进行手动验收的详细步骤和验证点。

---

## 验收项清单

### 1. 房主创建 waiting 房间
- [ ] 步骤：在浏览器中输入大厅地址，点击“创建并加入房间”。
- [ ] 期待结果：成功创建房间并分配 0 号座位，房间状态显示为“等待中”。

### 2. 手机 A 加入房间
- [ ] 步骤：手机 A 扫码或输入房间号加入房间。
- [ ] 期待结果：手机 A 成功分配 1 号座位，大厅显示手机 A 在线。

### 3. 手机 A 刷新页面
- [ ] 步骤：手机 A 刷新浏览器页面。
- [ ] 期待结果：控制台或房主界面上，手机 A 的状态短时间内显示为 `○ 重连中`，但**不应被立即踢出房间或释放座位**。

### 4. 手机 A 在 60 秒内恢复座位
- [ ] 步骤：手机 A 在刷新后 10 秒内加载完成。
- [ ] 期待结果：手机 A 成功同步回 1 号座位，状态恢复为 `● 在线`，且能看到当前大厅状态。

### 5. 手机 A 超时释放座位 (60s)
- [ ] 步骤：手机 A 再次断开连接（如关闭页面）并等待超过 60 秒。
- [ ] 期待结果：60 秒后，房主界面上手机 A 的座位被释放，显示为“等待玩家加入...”，且旧 token/sessionId 彻底失效。

### 6. 手机 A 主动退出 waiting 房间
- [ ] 步骤：手机 A 再次加入房间，然后在等待大厅中主动点击“退出并解绑房间”按钮。
- [ ] 期待结果：
  - [ ] 弹出确认提示：`确定退出房间吗？退出后座位会释放。`
  - [ ] 点击确认后，手机 A 的座位**立即释放**，客户端清除 localStorage 对应的 session 状态。
  - [ ] 其他玩家能够实时看到座位变空。

### 7. 启动一局游戏
- [ ] 步骤：大厅中满 4 人（或使用 fill-ai 填满 Bot），房主点击“开始游戏”。
- [ ] 期待结果：游戏顺利启动，各玩家分发到手牌。

### 8. 手机 B 对局中刷新页面
- [ ] 步骤：在游戏进行中，手机 B（例如 1 号位玩家）刷新页面。
- [ ] 期待结果：
  - [ ] 刷新期间，其他玩家看到 1 号位玩家状态为 `○ 重连中`。
  - [ ] 手机 B 页面重新加载后，成功恢复连接并展现**完整的、与此前完全一致的手牌和桌面状态**，没有卡死或漏牌。

### 9. 手机 B 对局中断网超过 30 秒 (系统托管)
- [ ] 步骤：手机 B 锁屏、切后台或断开网络，并等待超过 30 秒。
- [ ] 期待结果：
  - [ ] 30 秒后，其他玩家看到 1号位玩家状态变更为 `已离开 / AI 托管` 或 AI 标识。
  - [ ] 如果轮到 1 号位出牌，系统自动为其托管打出一张牌，并在日志栏记录 `[系统托管出牌] 玩家B离线`；如果是吃碰杠胡响应，系统默认选择“过”或有胡自动“胡”。
  - [ ] 牌局没有卡死，其他在线玩家可以正常操作。

### 10. 手机 B 恢复网络并接管托管
- [ ] 步骤：手机 B 在 120 秒内恢复网络并重新进入游戏。
- [ ] 期待结果：
  - [ ] 成功接管席位，状态恢复为 `● 在线`。
  - [ ] 托管状态解除，手机 B 可以继续控制后续操作。

### 11. 手机 B 离线超时 (120s)
- [ ] 步骤：手机 B 再次断网，并等待超过 120 秒。
- [ ] 期待结果：
  - [ ] 状态从 `reconnecting` 变更为 `● 离线` (offline)。
  - [ ] 席位仍然保留，托管打牌逻辑继续工作。

### 12. 手机 B 主动退出正在进行的牌局
- [ ] 步骤：手机 B 重连后，在对局中点击“离开”按钮。
- [ ] 期待结果：
  - [ ] 弹出确认提示：`对局已开始，退出后将由 AI 托管，是否确认？`
  - [ ] 确认后，连接彻底断开，客户端本地 session 被清理。
  - [ ] 其他玩家看到该席位变更为 `已离开 / AI 托管`，游戏继续运行。

### 13. 结算阶段安全离开
- [ ] 步骤：游戏决出胜负进入结算状态，玩家点击“返回联机大厅”按钮。
- [ ] 期待结果：
  - [ ] 弹出确认提示：`确定离开结算页面吗？`
  - [ ] 确认后，本地 session 立即清除并回到加入页。
  - [ ] 服务端结算数据保留，直到房间 TTL 清理。

### 14. 房间 TTL 垃圾清理
- [ ] 验证点：
  - [ ] 没有任何真人的 waiting 空房间，在 5 分钟后被服务端自动回收删除。
  - [ ] 仅剩 left（主动退出）或 AI 的对局房间，在 2 分钟后被自动回收删除。
  - [ ] 处于 settlement（结算）阶段 of 房间，在 5 分钟后被自动回收删除。
  - [ ] **有真人在线的房间永远不被清理**。

### 15. 暗牌隔离安全性
- [ ] 验证点：在游戏非结算阶段，通过任意客户端（哪怕是掉线重连的玩家）接收到的 `PlayerVisibleView` 数据中，**不含有其他在线/离线真人和 AI 的手牌明细**，避免通过 F12 面板偷看暗手牌。
