# 长沙麻将单机版 v0.4.1 可玩性手动验收测试用例

为确保浏览器版在真实玩家交互中的稳定性，请遵循以下流程进行手动验收。

## 验收流程说明

### 1. 初始化与开局
- [ ] 打开浏览器并进入游戏主页 `http://localhost:3000` (或控制台输出的对应 Vite 端口)。
- [ ] 观察配置区域：能自由调整扎鸟规则（开关/数量）、AI 性格。
- [ ] 点击 **“🀄 开始新局”** 按钮。
  - **预期**：配置面板隐藏，游戏桌面初始化，玩家发得 13 或 14 张手牌，开始播放对局。

### 2. 摸打基础交互
- [ ] 当轮到玩家本人出牌时（手牌区域高亮，且提示“等待您出牌”）：
  - [ ] 点击手牌中的一张牌进行选择（应出现向上弹起/高亮视觉）。
  - [ ] 点击另一个牌，前一张牌落下，后一张弹起（单选切换正确）。
  - [ ] 再次点击已选择的牌，该牌放下（取消选择正确）。
  - [ ] 点击 **“打出所选牌”** 按钮（或直接双击该牌）。
    - **预期**：手牌减少 1 张，牌河增加 1 张，打牌日志出现在右侧面板，且总牌数仍然守恒 108 张。

### 3. 重复/非法动作保护（防抖审计）
- [ ] **快速重复点击打牌**：在双击或打出牌的瞬间，极其快速地多次连续点击“打出”按钮。
  - **预期**：系统有内部锁保护，**绝对不会**产生多张扣减或触发状态错乱。
- [ ] **非法阶段出牌**：在 AI 摸打过程中，或者在等待胡/碰/吃响应的阶段（界面显示“等待其他玩家响应”），点击手牌并尝试打出。
  - **预期**：任何出牌指令均被静默拦截，不触发任何状态变化，手牌不减少。

### 4. 战术动作响应 (吃、碰、杠、胡、过)
- [ ] 观察 AI 出牌。当 AI 打出你的可用牌时，下方的战术操作面板动态滑出。
  - [ ] **验证按钮显示**：只有合法的动作（胡/碰/吃）按钮才被渲染。
  - [ ] **多吃法展示**：若该牌有多种吃牌组合（如打 4w，手牌有 2w3w/5w6w），点击“吃”按钮会展开多个具体选项子面板。
  - [ ] **点击“过”**：点击“过”后，按钮清空，AI 继续步进摸打。
  - [ ] **点击“吃/碰/杠”**：选择相应动作。
    - **预期**：手牌对应消耗，副露（Melds）展示区新增公开牌组，游戏继续进行，DebugPanel 的 `hands + melds + discards + wall + birdTiles` 精确等于 108。

### 5. 结算锁定与再来一局
- [ ] 推进对局直到有人胡牌或摸完所有牌流局。
  - **预期**：结算弹窗 (SettlementModal) 自动弹出，AI 推进完全静默锁定，对局中止。
- [ ] 在弹窗显示期间，尝试点击背景、点击打牌、或使用键盘等。
  - **预期**：主游戏桌面状态被完全锁定，无任何多余动作触发。
- [ ] 点击结算弹窗中的 **“再来一局”** 按钮。
  - **预期**：弹窗消失，当前局数据彻底清空，旧日志归档，全新发牌，开启新的一局，Debug 审计牌数再次重置为 108 守恒。

### 6. 持续稳定性（压力测试）
- [ ] 连续不间断玩满 **3 局** 游戏（包含胡牌和流局）。
  - **预期**：
    - 控制台 (`F12`) 无任何 `Error` 或 `Unhandled Promise Rejection`。
    - 页面响应流畅，AI 行动不卡死、不无限循环。
    - 每次结算和新局，DebugPanel 的总牌数审计恒为 108。
