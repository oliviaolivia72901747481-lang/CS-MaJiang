# 长沙麻将多人联网版 v0.8.4 手机端体验验收 Checklist

本 Checklist 指引开发者与测试人员对手机联网端进行真机/模拟器测试。

## 一、联机准备与大厅加入
- [ ] 房主运行 `npm run dev:online` 或 `npm run dev` 启动本地测试服务器。
- [ ] 真机 A (作为房主) 通过扫码或输入局域网 IP/端口进入游戏大厅。
- [ ] 真机 B (作为玩家) 通过扫码 IP/端口或在大厅列表中找到相同房间加入。
- [ ] 房主页面实时正确显示已加入玩家数量。

## 二、2 人真人局开局与对局 UX
- [ ] 选择 2 人模式并点击“开始游戏”（不勾选补满 AI 机器人）。
- [ ] 2 人局顺利开局，座位固化在活跃席位，并且发牌完全符合 14/13 张。
- [ ] **手机端只显示 1 个对手卡片（非活跃的 2 个席位已被完全隐藏）。**
- [ ] 页面无任何横向滚动条，宽度完全自适应在 100vw 内。
- [ ] 对局日志、诊断面板、帮助信息默认处于折叠隐藏状态。

## 三、手牌点按与操作 UX
- [ ] 玩家手牌尺寸合理，在一行或两行内完全展示，无挤出或重叠。
- [ ] 选中某张手牌后，该手牌有明显的**上浮 (translateY(-12px)) 和黄色高亮发光阴影**。
- [ ] 再次点击已选中的手牌，能安全稳定地打出该手牌。
- [ ] 摸到最新牌时，最后一张牌右侧产生 **12px 间距的物理隔离**，且上方有黄色的“**摸**”字角标。
- [ ] 弃牌河有横向滑动摘要展示，且最新被打出的弃牌有高亮显示。

## 四、动作面板触控尺寸 (MobileActionBar)
- [ ] 当出现吃碰杠胡动作选择时，弹出动作条。
- [ ] “过”按钮和“胡/碰/吃/杠”按钮高度均不低于 44px，且有充足的间距（避免误触）。
- [ ] 胡牌按钮 (huAction) 拥有红色流光渐变背景，且伴随持续呼吸灯动画特效。
- [ ] 在动作点击等待期间 (actionPending)，动作条按钮全部变灰禁用，防止玩家多次点击发送重复请求。
- [ ] 当有多种吃法/杠法时，屏幕中央弹出吃/杠选项小卡片，支持精确指点。

## 五、回合变化、日志与连接提示 (CurrentTurnBanner & Badges)
- [ ] 顶部提示栏 (CurrentTurnBanner) 能够清晰跟随出牌权变化：
  - [ ] 轮到自己出牌：显示闪烁的黄底“轮到你出牌”。
  - [ ] 轮到对手出牌：显示灰底“等待 玩家B 出牌”。
- [ ] 最近的出牌/吃碰动作会引发一个自动淡出的 toast 浮层 (RecentActionToast)，不遮挡我的手牌和按钮。
- [ ] 模拟网络断开：
  - [ ] 自己断线：页面中央出现遮罩“重连中... 正在恢复牌局”。
  - [ ] 对手断线：对手卡片连接状态显示“○ 重连中...”。
- [ ] 超时 30s 触发托管：对手卡片状态变为“已离开 (AI托管)”，机器人头像变为 🤖 并代为迅速出牌。

## 六、结算展示与重置 (SettlementModal)
- [ ] 2 人局结算弹窗弹出时：
  - [ ] **只显示 2 个活跃玩家的结算得分与手牌**，隐藏另外 2 个不活跃席位。
  - [ ] AI 玩家右侧标有 🤖 AI 标识。
  - [ ] delta 变动得分采用红绿区分，正数加分，负数减分。
  - [ ] 底部“返回联机大厅”按钮触控面积充足，且点击后确认退出。

## 七、全量回归与部署
- [ ] 运行 `npm test`，734 个测试用例全部 100% 通过。
- [ ] 运行 `npm run build`，编译完全无错。
