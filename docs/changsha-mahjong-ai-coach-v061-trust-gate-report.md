# 长沙麻将 AI 陪练可信度质量门 v0.6.1 报告

本阶段对 AI 陪练助手的所有实时评估、出牌建议、防守风险判断、有效进张计数及局后复盘进行全面审计和重构，修复了所有可能导致“偷看”或使用玩家不可见信息的漏洞，建立了严密的玩家视角可见信息模型，并补充了测试用例。

## 一、发现并修复的问题

1. **信息透明性 (偷看牌墙与手牌漏洞)**：
   - *发现*：原有效进张的计数与风险判断，在传入 `GameState` 时可以直接访问 `wall` 和 `state.players[1,2,3].hand`（暗手牌）。即使代码表面没有主动使用，依然缺乏强制的安全屏障。
   - *修复*：新增并强制使用 `VisibleStateForCoach` 物理层隔离。实时提示模块只允许消费 `VisibleStateForCoach` 的数据。

2. **有效进张估算不准确**：
   - *发现*：原进张剩余张数可能在文案中给玩家一种“绝对掌控”的误导（例如：“精确剩余 X 张”）。
   - *修复*：重写了 `getRemainingTileCount`。计算公式严格限定为：`理论剩余 = 4 - 玩家自己手牌 - 所有公开牌河 - 所有公开副露`，并在 UI 显式更新为“玩家视角估计/理论剩余”。

3. **风险判断含有绝对词**：
   - *发现*：原风险判定常使用“放铳风险极高”、“绝对高风险生张”等词汇，容易误导玩家。
   - *修复*：替换所有绝对性词汇（删除“绝对”、“一定”等），替换为概率化描述（“可能”、“从已公开牌河推测”）。

4. **局后复盘混淆信息来源**：
   - *发现*：原有复盘报告无法区分当时决策时的可见信息与结算后的全局结果。
   - *修复*：在 `ReplayReport` 中新增 `ReplayInsight`（深度诊断）字段，将分析分为 `realTimeKnown`（【当时可知】）和 `afterTheFact`（【事后观察】）两类。

5. **出牌推荐与手牌的一致性**：
   - *发现*：理论上存在极低概率推荐了手牌中已打出或未持有的牌。
   - *修复*：在 `recommendDiscards` 强加过滤，确保 Top-3 候选必须在玩家当前持有的手牌实例中，且去重。
