# 长沙麻将交互体验与视觉优化 v0.5 交付报告

本阶段专注于提升单机浏览器版本的操作手感、视觉沉浸感和规则清晰度。通过引入 3D 骨质麻将视觉效果、中心罗盘指示器、AI 个性化徽章、迷你牌型吃碰杠按钮、海底结算明细与桌面最终复盘等升级，使游戏原型体验更为完整逼真。

---

## 一、已优化内容清单

### 1. 麻将牌视觉升级 (Mahjong Tile Upgrades)
- **3D 材质**：牌面采用优雅的象牙白（Ivory Face）渐变，搭配 3D 悬浮投影与边缘倒角；牌背采用深邃的祖母绿（Emerald Green）。
- **花色区分**：万（红色）、筒（蓝色）、条（绿色）花色字符醒目，契合真实麻将色彩感受。
- **动态悬浮**：可点击的牌在 Hover 时自动上浮 4px 并加高亮描边；已选中牌上浮 8px 并附带金黄发光呼吸阴影，提升确认感。
- **最新弃牌指示**：最近打出的弃牌会增加橙色小圆点标记与微弱呼吸灯效果，使用户极易捕捉最新的打牌动态。
- **禁用态区分**：非本人出牌或锁定阶段的手牌会降低透明度，禁用任何点击响应。
- **公开翻牌**：结算时，全家手牌（包括 AI 的暗手牌）自动从牌背状态翻开公开，让胜负一目了然。

### 2. 桌面布局与状态指示 (Desk Layout & State Indicators)
- **中心方向罗盘 (TurnIndicator)**：去除了文字版的简易状态，在牌桌中央设计了精致的数字罗盘。
  - **轮值指向**：亮黄色指针动态指向当前出牌方位（南 = 我，东 = 下家，北 = 对家，西 = 上家）。
  - **庄家徽记**：庄家座位上方带有红色的 “庄” 标志。
  - **局势概要**：正中实时显示“剩余牌墙数”，并在底部渲染当前子阶段（发牌、起手胡、摸打、海底、结算等）。
- **AI 个性 badge (AiProfileBadge)**：AI 昵称旁边高亮展示其出牌倾向：
  - ⚖️ 均衡型 (balanced)
  - 🚀 快胡型 (fastHu)
  - 👑 大胡型 (bigHu)
  - 🛡️ 防守型 (defensive)
- **Utility 布局**：牌河与手牌合理分配栅格，避免发生任何重叠与挤压。

### 3. 出牌与操作按钮体验 (Player Actions & ActionPanel)
- **双击快捷出牌**：支持选中牌直接“双击”快速打出，或通过控制面板确认“打出选中牌”，简化频繁的出牌交互。
- **按钮分层与分组**：
  - **胡牌组**：胡/自摸按钮拥有最高的红色渐变发光优先级。
  - **副露组**：碰/吃/杠归入中性蓝绿色组，并且多种吃法以顺子组合（如 `3万 + 4万`）的小骨面 mini-tiles 直观呈现在吃牌按钮内，摆脱文字阅读成本。
  - **过牌**：低调的灰色按钮，悬浮时有气泡说明。
  - **一击防 race**：任何动作一经点击，立即进入 lock 忙碌锁阶段并全部 disable 动作列表，防止二次重复误触破坏对局流转。

### 4. AI 行动实时反馈 (AI Action Notification)
- **最近动作 Toast (LastActionToast)**：AI 摸打牌或执行吃碰杠后，牌桌正中上方弹出微悬浮通知（如 `⚡ 【机器人 2】碰牌：5筒`），并在 3 秒后自动淡出，让玩家不看右侧日志也能跟上对局节奏。

### 5. 结算与复盘机制 (SettlementModal & Final Review)
- **海底结算**：支持按赢家/自摸/点炮输出醒目的插画风格徽章。
- **玩家名称反查**：结算明细中，原始晦涩的 Player UUID 被完全翻译反查为友好的 `玩家本人` / `机器人 x` 名称，使积分流动一目了然。
- **扎鸟双击命中**：扎鸟显示牌面，且命中（Hit）的鸟牌呈绿色靶心效果，未命中的呈红色叉，标注中鸟玩家。
- **关闭后复盘**：结算弹窗可以点击“关闭并查看桌面牌局”，此时弹窗收起，桌面上依然保持终局的手牌、副露、弃牌状态。桌面上方提供悬浮复盘快捷栏，允许玩家再次唤起“查看结算详情”或启动“再来一局”。

### 6. 日志分类与去重 (Game Log Enhancements)
- **色彩分类标签**：出牌（灰色）、摸牌（深蓝）、碰/吃/杠（蓝绿）、胡（红色）、系统（青色），极易扫视过滤。
- **思考内容拆分**：AI 的打牌理由（如 `防守阶段，优先打出安全牌`）与打出细节（如 `8条`）完全剥离，思考理由单独作为浅灰色小字排版在下方，主日志行保持干净。
- **日志长度封顶**：超过 100 条时自动清除头部历史日志，确保大容量网页运行流畅。

### 7. 规则配置面板 (RuleConfigPanel Upgrades)
- **参数控制**：新增了对 `scoreMode` (6/7 分制 vs 6/6 分制) 以及 `openDoor.needOpenDoorForDianPaoHu` (点炮胡是否需要玩家已吃碰杠开门) 的设置。
- **开局锁定**：只要本局游戏处于进行中，配置面板会在顶部呈现“🔒 已锁定”标示，且所有选项表单强制置为 `disabled` 态，避免中途修改规则触发引擎状态机分裂。

## 四、手动验收指南

运行以下命令启动开发环境：
```bash
npm run dev
```
打开网页后，手动流程如下：
1. **规则设置**：点击“⚙️ 长沙麻将玩法配置”展开，切换“6/6 分制”与“开门限制”等，配置三家 AI 的性格（如大胡型、防守型）。
2. **开局**：点击“开始新局”。规则面板会自动锁定，剩余牌墙数初始化。
3. **摸打**：
   - 鼠标悬停在手牌上，牌应轻微上浮；点击一张，该牌浮起并伴随金色呼吸阴影，上方提示牌名（如 `👉 已选中: 三万`）。
   - 双击该牌即可打出。AI 打牌时会有 500ms 延迟，并在上方弹出浮动 Toast 描述其最新动作。
4. **决策**：当 AI 点炮或打出可以吃碰的牌时，ActionPanel 滑出。可以看到“吃”按钮里渲染了小巧的迷你骨牌。点击“过”进入下一步。
5. **终盘复盘**：对局结束后，结算弹窗弹出。可以看到 UUID 已替换为昵称，鸟牌显示了中鸟结果。点击“关闭并查看桌面牌局”，弹窗消失，对局残留，页面上方悬浮复盘栏。点击“再来一局”重置游戏。
